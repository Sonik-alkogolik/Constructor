# Инструкция по использованию Image to Code

## 🚀 Быстрый старт

### 1. Запуск приложения
```bash
npm run dev
```
Откройте http://localhost:3000 в браузере.

### 2. Загрузка макета
1. Нажмите кнопку **"🖼️ Из макета"** в верхней панели
2. Перетащите PNG/JPG изображение в зону загрузки или кликните для выбора файла
3. Дождитесь превью изображения

### 3. Анализ макета
1. Нажмите кнопку **"🚀 Анализировать макет"**
2. Дождитесь завершения анализа (прогресс бар покажет этапы):
   - 📐 Анализ структуры макета
   - 📝 Распознавание текста (OCR)
   - 🎨 Извлечение цветовой палитры
   - ⚙️ Генерация HTML/CSS

### 4. Просмотр результатов
После анализа откроются вкладки с результатами:

#### 📊 Структура
- Размер макета (ширина × высота)
- Количество найденных областей
- Список регионов с размерами и позициями

#### 🎨 Цвета
- Цветовая палитра (до 8 цветов)
- Основной (доминантный) цвет помечен бейджем
- HEX значения для каждого цвета

#### 📝 Текст
- Распознанные текстовые строки
- Процент уверенности распознавания
- Цветовая индикация качества:
  - 🟢 Зелёный (>80%) - отлично
  - 🟡 Жёлтый (50-80%) - хорошо
  - 🔴 Красный (<50%) - низкое качество

### 5. Применение кода
1. Просмотрите сгенерированный код во вкладке "📝 Сгенерированный код"
2. Нажмите **"✅ Применить код в редактор"**
3. Код автоматически вставится в HTML и CSS редакторы
4. Закройте модальное окно кнопкой "Отмена" или кликом вне окна

## 💡 Советы для лучших результатов

### Подготовка макета в Figma
1. **Экспортируйте в высоком качестве** - используйте PNG для чёткости текста
2. **Убедитесь в читаемости текста** - контрастный текст на фоне
3. **Простая структура** - вертикальные макеты работают лучше
4. **Минимум наложений** - элементы не должны перекрывать друг друга

### Ограничения
- **Сложные сетки** могут распознаваться неточно
- **Перекрывающиеся элементы** могут быть объединены
- **Декоративные элементы** могут быть пропущены
- **Иконки и SVG** не распознаются отдельно

---

## 🔧 Решение проблем

### Проблема: Низкое качество OCR
**Решение:**
- Убедитесь, что текст на макете чёткий
- Используйте PNG вместо JPG
- Увеличьте размер экспортируемого макета

### Проблема: Долгий анализ
**Решение:**
- Уменьшите размер изображения (оптимально 1920×1080)
- Закройте другие вкладки браузера для освобождения памяти

### Проблема: Код не применяется
**Решение:**
- Проверьте консоль браузера (F12) на ошибки
- Убедитесь, что анализ завершён успешно (100% прогресс)
- Попробуйте перезагрузить страницу

## 📊 Как это работает

### Архитектура
```
Загрузка изображения
       ↓
Tesseract.js (OCR) → Распознавание текста
       ↓
ColorThief → Извлечение палитры
       ↓
Canvas Analysis → Поиск областей
       ↓
Генерация HTML → Структура дерева
       ↓
Генерация CSS → Стили + адаптивность
       ↓
Применение в редактор
```

### Используемые технологии
- **Tesseract.js** - распознавание текста (русский + английский)
- **ColorThief** - извлечение цветовой палитры
- **Canvas API** - анализ структуры изображения
- **Vue 3** - реактивный интерфейс
